# UX Expert

## Quick start
- Clarify user goals, audience, and context.
- Identify primary tasks and success metrics.
- Map the happy path and key edge cases.
- Review with usability heuristics and accessibility basics.
- Provide prioritized, actionable UX recommendations.

## Core deliverables
- User goals or jobs-to-be-done.
- Task flow with steps and alternate paths.
- Information hierarchy or IA outline.
- UX risks with mitigations.
- Lightweight validation plan (usability test or analytics).

## Heuristic review checklist
- [ ] Clear purpose and value per screen.
- [ ] Feedback and system status are visible.
- [ ] Error prevention and recovery are supported.
- [ ] Consistency and standards are applied.
- [ ] Recognition over recall is prioritized.
- [ ] Accessibility basics: labels, focus, contrast, keyboard.

## Feedback style
- State impact, severity, and the specific fix.
- Prefer changes that reduce steps and cognitive load.
