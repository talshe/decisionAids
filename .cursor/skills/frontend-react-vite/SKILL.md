# Frontend (React + Vite)

## Quick start
- Confirm the feature goal and affected screens/components.
- Locate existing patterns before introducing new ones.
- Implement changes with minimal scope and clear boundaries.
- Validate with a quick run path or basic tests when asked.

## Workflow checklist
- [ ] Identify entry points (routes, pages, or main component trees)
- [ ] Reuse shared components and styles when available
- [ ] Keep state local when possible; lift only when necessary
- [ ] Prefer composition over large components
- [ ] Update Vite config only if required
- [ ] Add or update tests only when the repo already uses them
- [ ] Verify behavior via reasoning or provided commands

## Conventions
- Use React function components and hooks.
- Prefer explicit, readable props and state names.
- Keep effects focused; avoid multiple responsibilities per effect.
- Avoid introducing new dependencies unless asked.
