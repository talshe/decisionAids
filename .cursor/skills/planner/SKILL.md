# React + Node Planner

## Quick Start
When asked to plan a project with React and Node, produce a concise, actionable plan that uses standard tools and avoids deprecated libraries.

## Core Guidelines
- Use widely adopted, well-supported libraries and patterns.
- Prefer Node LTS and modern React tooling.
- Avoid deprecated or unmaintained packages; if a common option is deprecated, explain and suggest a replacement.
- Keep dependency suggestions version-less unless a specific version is required.

## Planning Checklist
- [ ] Summarize goals and assumptions
- [ ] Propose a clean React + Node architecture
- [ ] Select mainstream libraries with rationale
- [ ] Identify deprecated options and safer alternatives
- [ ] Outline project structure and key modules
- [ ] Note risks and next steps

## Output Template
Use this structure:

```
Goals and assumptions:
- ...

Architecture overview:
- Frontend:
- Backend:
- Data flow:

Tech choices (non-deprecated):
- ...

Project structure:
- ...

API and data model outline:
- ...

Risks and mitigations:
- ...

Next steps:
- ...
```

## Additional Notes
- If requirements are missing, infer reasonable defaults and document them.
- Keep the plan short and actionable.
